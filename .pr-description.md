# Fix: Bug batch — security, correctness, and robustness hardening

## Summary

Three rounds of fixes addressing **24 distinct bugs** found during a thorough
audit of the shell scripts and WebUI source. The most severe issue was a
broken flash guard in `boot_patch.sh` that could attempt to flash a
non-existent `new-boot.img` on block-device bootimages (silent bricked
device risk). Several shell-injection vectors were also closed.

## Test plan

- [x] `bash -n` syntax check on all modified shell scripts
- [x] `vite build` clean (3 runs)
- [ ] Manual smoke test on a real device (post-merge)
- [ ] `validate_module.js` — 5 environmental failures (missing downloaded
      binaries) unrelated to this PR

## Impact

- **15 files, +289 / -101** across 4 commits
- **4 new behaviors** the user can verify:
  1. Patch flow: flash guard now refuses to write if `new-boot.img` is missing
  2. KPM repo: downloads capped at 50 MiB
  3. Rehook switch: rolls back UI on backend failure
  4. Pull-to-refresh: ignores multi-finger gestures

## Bug list

### 🔴 P0 — Critical (commit 85cbc1a)

| # | File | Bug | Fix |
|---|---|---|---|
| 1 | `module/patch/boot_patch.sh` | Flash guard `if [ -b ] \|\| [ -c ] && [ -f ]` parsed as `[ -b ] \|\| ( [ -c ] && [ -f ] )` — block-device bootimages would **always** attempt to flash even when `new-boot.img` missing | Explicit nested `if` test |
| 2 | `module/patch/boot_patch.sh` | `$?` after `magiskboot repack` actually read the `echo` that came after it, not magiskboot | Replaced with `if ! magiskboot ...` |
| 3 | `module/patch/boot_patch.sh` | First `-M <file>` was silently skipped by KPM validator (prev_flag empty on iteration 0) | Initialize to sentinel `__start__` |
| 4 | `module/patch/boot_patch.sh` | Bogus `b700` big-endian aarch64 check (ELF is always LE) | Dropped |
| 5 | `module/service.sh` | Subshell-piped `while` loop lost state on exit; CSV parse used unquoted `$pkg`/`$uid` | Switched to here-doc reader, 5-min `boot_completed` timeout, sed-escaped package name |
| 6 | `module/compile_kpm.sh` | Compiler exit code masked — `$?` read the subsequent `[ "$ARCH" = ... ]` test | Explicit `compile_rc=$?` capture |
| 7 | `module/install_kpm.sh` | `find -o` precedence bug — accidentally matched `.kpm` directories and `.bak` files | Added explicit `-type f \( -o -o \)` |
| 8 | `module/install_kpm.sh` | `$ARGS_OPT` unquoted in `kpatch kpm load` — broke on args with spaces | Quoted |
| 9 | `build.sh` | `pnpm build \|\| pnpm install && pnpm build` parsed as `(pnpm build \|\| pnpm install) && pnpm build` | Wrapped in `{ }` |
| 10 | `webui/page/kpm.js` | User-controlled `file.name` interpolated into `loadModule` and `cp` shell commands — shell injection | `sanitizeFilename` once + `escapeShell` on all path concats |
| 11 | `webui/page/kpm_repo.js` | Custom repo URL from `localStorage` passed straight to `curl` shell command | `sanitizeUrl()` (http/https only) |
| 12 | `webui/page/exclude.js` | CSV written via `echo "${csvContent}"` — package name with `"` `$` `` ` `` would inject | Single-quoted heredoc |
| 13 | `webui/page/backup.js` | `backup.name` from `ls` passed to `cp` target without stripping path components — could write outside `/storage/emulated/0/Download/` | Strip `..` and path separators |
| 14 | `webui/index.js` | Rehook switch flipped optimistically; no rollback on backend failure | Re-fetch status on failure |
| 15 | `webui/index.js` | `initInfo` rendered literal `"undefined"` when `uname` returned empty | Em-dash fallback |
| 16 | `webui/index.js` | `setRehookMode` shell command had unquoted `${persistDir}/rehook` and `${modDir}/status.sh` | `escapeShell` |
| 17 | `webui/page/log.js` | `tail -200` — toybox interprets as byte offset, not line count | `tail -n 200` |
| 18 | `module/status.sh` | `cat "$file.tmp" > "$file"` race window with concurrent readers; sed broke on `/` in values | `mv` and `\|` delimiter |

## Notes

- The P0 list comes from a static audit using `explore` agents and manual
  re-verification of each claim against source. Backup branch
  `backup-pre-bugfix` preserves the pre-fix state.
- One incidental commit (`682b478`) removed an accidentally-tracked
  `ksu_log.tar.gz` from the working tree.
- The shell injection vectors (#10-13) all required root (kernelsu-alt
  exec) to exploit but were nonetheless trivial to close.
- The flash guard bug (#1) was the highest-impact issue: a user on
  block-device boot images could see a "Flashing" status even when the
  repack step had silently failed.
